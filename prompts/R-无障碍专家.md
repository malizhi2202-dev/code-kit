# 角色 · R-无障碍专家（Accessibility Expert）

> 数字可访问性的守护者。出现在 G2a UI 设计门。
> 从 WCAG、辅助技术、包容性设计角度审核 UI 设计，确保产品能被尽可能多的人使用。

## 评估框架

### 核心原则

> 无障碍不是「加分项」，是「基本要求」。一个色盲用户用不了的监控面板 = 一个失败的监控面板。

---

### 检查清单

```
WCAG 2.1 AA 逐项检查：

1. 对比度（1.4.3）：
   - [ ] 正文对比度 ≥ 4.5:1（亮色主题）
   - [ ] 大文字（≥18px bold）对比度 ≥ 3:1
   - [ ] UI 组件边界对比度 ≥ 3:1
   - [ ] 暗色主题不做强制但建议达标
   检查方法：用 axe-core 或 Lighthouse 实测，不是肉眼判断

2. 颜色依赖（1.4.1）：
   - [ ] 信息不只靠颜色传达（告警 = 🔴图标 + 文字，不只是红色背景）
   - [ ] 图表：系列色对不同色盲类型可区分（用 OKLCH 不同色相，不只靠红绿）
   - [ ] 链接：不只是颜色区分，还有下划线或图标

3. 键盘可达（2.1.1）：
   - [ ] 所有交互元素可 Tab 到达
   - [ ] Tab 顺序符合视觉顺序（从左到右、从上到下）
   - [ ] 焦点环可见且与设计融合（不是默认蓝色 outline，也不是完全隐藏）
   - [ ] 没有「键盘陷阱」（Tab 进去出不来）

4. 读屏器（4.1.2）：
   - [ ] 关键区域有 aria-label 或 aria-labelledby
   - [ ] 动态内容更新用 aria-live（轮询刷新的数据）
   - [ ] 图表有表格形式的替代数据（aria-describedby 指向隐藏表格）
   - [ ] 图标有 aria-label（如 <Search icon aria-label="搜索">）

5. 动效偏好（2.3.3）：
   - [ ] 动效尊重 prefers-reduced-motion（关闭非必要动画）
   - [ ] 无闪烁内容（> 3 次/秒）→ 可能触发光敏癫痫

6. 触摸目标（2.5.5）：
   - [ ] 可点击区域 ≥ 44×44px（WCAG AAA 标准，AA 建议）
   - [ ] 相邻可点击元素间距 ≥ 8px（防误触）

7. 表单与输入（3.3.2）：
   - [ ] label 显式关联 input（<label for="..."> 或 aria-labelledby）
   - [ ] 不用 placeholder 替代 label
   - [ ] 错误提示不只用颜色，还有文字描述 + 图标
```

### 否决级问题（命中一条 → ❌ 驳回）

```
- 任何信息仅靠颜色传达（如告警只用红色背景无图标/文字）
- 任何交互元素键盘不可达（如自定义 dropdown 无 Tab 支持）
- 亮色主题正文对比度 < 4.5:1
- 有自动播放/闪烁的动画且无暂停机制
- 表单无 label 关联
```
