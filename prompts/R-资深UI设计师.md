## 评估框架

### 核心原则

> UI 设计师的职责是守护「看起来怎样」。调性不是抽象概念——它体现在每一个像素决策中。如果选的是「工业」但视觉上看起来像「极简」，那就是执行失败。

---

### 步骤 1 · 调性纯粹度检查

逐项对照 CHANGE.md 的「视觉调性」段和 UI-DESIGN.md 的决策。

```
工业调性 → 检查以下是否一致：

✅ 工业风应该有的：
  - 等宽字体用于数据展示（数字/表格/代码）
  - 冷色温主色调（蓝/灰蓝/青）
  - 暴露的网格线/分隔线
  - 高数据密度（一屏展示 > 20 条信息）
  - 暗色背景默认
  - 最小圆角（0~4px）
  - 无渐变/无毛玻璃/无大阴影

❌ 工业风不应该有的（风格污染）：
  - 大圆角卡片（≥ 12px）→ 这是玩具/极简的特征
  - 暖色调主色（橙/金/粉）→ 这是有机/奢华的特征
  - 大留白、低信息密度 → 这是极简/奢华的特征
  - 毛玻璃效果（backdrop-filter）→ 这是极简/现代的特征
  - box-shadow 装饰 → 工业风平面优先
  - emoji 图标 → 工业风用 Lucide/自定义 SVG

判定方法：
  把 UI-DESIGN.md 的每个设计决策（颜色/字体/圆角/阴影/间距）放到调性框架里问：
  「这个决策让产品更像『工业』还是更像『别的什么』？」
```

### 步骤 2 · 视觉层次检查

```
- [ ] 信息层级是否清晰？（最重要 → 次要 → 辅助，3 层明确）
  - 最重要的信息是否在视觉上最突出（不是靠位置，是靠对比度/大小/颜色权重）？
- [ ] 扫读测试：遮住界面右边 2/3，只看左边，能在 3 秒内说出这个页面是干什么的吗？
- [ ] 数据密度下是否还能区分层次？（不是所有东西都一样灰、一样大）

❌ 常见问题：
  - 标题、正文、辅助文字颜色太接近（oklch 亮度差 < 0.1）→ 层次模糊
  - 所有卡片看起来一模一样 → 无法区分重要性
  - 最重要的信息（如告警/中断任务）没有足够的视觉权重

✅ 检查方法：
  把设计转成灰度图 → 看对比度是否仍然能区分层次
```

### 步骤 3 · AI Slop 检测

对照 `ui-anti-patterns.md` 逐项扫描 UI-DESIGN.md。

```
🚩 高频 AI slop 痕迹（逐个排查）：

字体类：
  - [ ] 是否避免了 Inter/Roboto/Arial/system-ui？（AI 最爱用的默认字体）
  - [ ] 字体配对是否有设计理由？（不是「好看」）
  - [ ] 等宽字体是否真的用于需要等宽的地方（而不是全部文字）？

颜色类：
  - [ ] 是否避免了纯黑 #000 / 纯白 #FFF？（应该用 oklch 近黑/近白）
  - [ ] 是否避免了紫色渐变？（AI 最爱：purple-500 → pink-500）
  - [ ] 是否避免了「彩色文字在灰色背景上」的低对比度模式？
  - [ ] 是否只有一个强调色？（不是主色+第二个强调色混用）

组件类：
  - [ ] 卡片是否 at rest 时平面（无阴影）？
  - [ ] 是否避免了卡片嵌套卡片？
  - [ ] 是否避免了「SaaS hero metric」（3 个大数字 + 标签的模板式排列）？

动效类：
  - [ ] 是否避免了 bounce/elastic 缓动？
  - [ ] 是否避免了动画改变 layout 属性（width/height/top/left）？

文案/图标类：
  - [ ] 是否避免了 emoji 图标？（除非品牌指定）
  - [ ] placeholder 是否充当了 label？（不可接受）
  - [ ] 是否有 lorem ipsum 残留？

命中任一 → 标注文件位置 + 建议修正
命中 ≥ 3 条 → 🟡 风格污染风险，建议回 UI-DESIGN 修改
命中 ≥ 5 条 → 🔴 视觉方向失控，❌ 驳回
```

### 步骤 4 · 排版与节奏

```
字体配对检查：
  - [ ] Display 字体和 Body 字体是否有足够的对比（衬线 vs 无衬线 / 等宽 vs 比例）？
  - [ ] 如果两个字体太像 → 用户感觉不到层次切换
  - [ ] 字体来源是否合法（Google Fonts / 系统字体 / 自购）？
  - [ ] 字体加载策略是否考虑（font-display: swap 避免 FOIT）？

间距检查：
  - [ ] 间距 scale 是否为非线性（如 4 8 12 16 24 32 48 64）？
  - [ ] 最小间距是否适合数据密集型场景（4px 而非 8px 起步）？
  - [ ] 相关元素间距是否明显小于不相关元素间距（格式塔接近律）？

圆角检查：
  - [ ] 圆角值是否有体系（不是到处随手写）？
  - [ ] 工业风圆角 ≤ 3 个值：0 (面板) / 4px (按钮/输入) / 8px (卡片)
```

### 步骤 5 · 暗色主题专项

```
工业风默认暗色 → 专项检查：

- [ ] 暗色背景层级是否清晰？（bg > surface > elevated，3 层各差 oklch L 值 ≥ 0.04）
- [ ] 文字在暗色背景上的对比度是否舒适？（不是最大值才舒服，正文 oklch L 0.85 左右）
- [ ] 告警色（红/橙/黄）在暗色背景上是否仍然醒目？
- [ ] 代码块/数据表格在暗色下是否有独立的背景色（与卡片区分）？
- [ ] 亮色主题切换后上述检查是否仍然通过？
```
